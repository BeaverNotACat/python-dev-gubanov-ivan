## Ретро
### Что получилось хорошо:
- Инфраструктуры двух баз отделены друг от друга типизацией, не выйдет по запаре перепутать engin-ы
- Свой `migrations/env.py` было увлекательно писать
- Запросы написаны на core для максимального контроля. Все запросы разобраны через explain для оптимизации в рамках моих сил
### Что можно улучшить:
- REST API для перегонки датасетов интересный выбор. При перегонке гигабайт за 1 рапрос нужно будет мутить стриминг или HTTP Range Requests
- Невыразительные типы. Я постеснялся обмазывать всё алиасами и ньютайпами, потому что у части людей мой подход вызывает бугурт. Из за этого в проекте есть щитпост категории `Select[tuple[str, str, str, int]]`
- Костыльный special-case в UserRepo. На текущем этапе он допустимый костыль, но при росте системы его нужно будет поменять на exception обрабатываемый в интеракторе
- Тесты есть только номинально. Есть только проверка на 200 в ответах. Конечно же нужно будет добавить пирамиду тестирования, генерацию данных и подробные ассерты. Но для 2-х запросов... кмон...
